## Overview

The Google Play Console adapter provides comprehensive access to the **Google Play Developer API** for Android apps. It uses OAuth 2.0 with long-lived refresh tokens for user authorization, and the **Edits API** pattern — all content changes are staged in a draft "edit" and committed atomically. Real-time notifications are delivered via **Google Cloud Pub/Sub**.

**Capability:** [`app-store`](/docs/capabilities/app-store)

**Auth model:** OAuth 2.0 + refresh tokens

**Token lifetime:** 1 hour, refreshable

---

## Prerequisites

1. Go to [Google Cloud Console](https://console.cloud.google.com) → **APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID**
2. Select **Web application**, add an **Authorized redirect URI** (e.g. `https://yourapp.com/oauth/callback/google-play`)
3. Copy **Client ID** and **Client Secret**
4. Enable the **Google Play Android Developer API** in the Cloud Console
5. In [Google Play Console](https://play.google.com/console): **Setup → API access → Link to Google Cloud project**, then grant your OAuth app access to the Play Console account
6. For real-time notifications: Set up a **Cloud Pub/Sub** topic and push subscription at `https://yourapp.com/webhooks/google-play`

## Registering with the hub

```typescript title="lib/integrations.ts" lineNumbers
import { IntegrationHub } from '@apollo-deploy/integrations';
import { createGooglePlayAdapter } from '@apollo-deploy/adapter-google-play';

const hub = new IntegrationHub();

hub.register('google-play', createGooglePlayAdapter({
  clientId: process.env.GOOGLE_CLIENT_ID!,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
  webhookSecret: process.env.GOOGLE_PLAY_WEBHOOK_SECRET,
}));

await hub.initialize();

hub.onEvent('google-play', (event) => {
  if (event.eventType === 'subscription.renewed') {
    const data = event.data as any;
    console.log('Subscription renewed:', data.subscriptionNotification?.purchaseToken);
  }
});
```

## OAuth flow

### Step 1 — Generate the authorization URL

```typescript title="lib/google-play-oauth.ts" lineNumbers
const adapter = hub.getAdapter('google-play');

const url = adapter.oauth.getAuthorizationUrl({
  state: crypto.randomUUID(),  // Store in session for CSRF
  scopes: ['https://www.googleapis.com/auth/androidpublisher'],
  redirectUri: 'https://yourapp.com/oauth/callback/google-play',
  extras: {
    access_type: 'offline',  // REQUIRED to get a refresh token
    prompt: 'consent',       // REQUIRED to get refresh token on re-auth
  },
});
```

<Callout type="warn">
Always pass `access_type: 'offline'` and `prompt: 'consent'` in `extras`. Without these, Google will not return a refresh token — or will only return one on the **first** authorization. Re-authorization without `prompt: 'consent'` will return an access token only.
</Callout>

### Available scopes

| Scope | Description |
|-------|-------------|
| `https://www.googleapis.com/auth/androidpublisher` | Full access to Google Play Developer API |

### Step 2 — Exchange the code

```typescript title="app/oauth/callback/google-play/route.ts" lineNumbers
const tokens = await adapter.oauth.exchangeCode({
  code: searchParams.get('code')!,
  redirectUri: 'https://yourapp.com/oauth/callback/google-play',
});

// tokens.accessToken    — Bearer token (1 hour)
// tokens.refreshToken   — Long-lived refresh token
// tokens.expiresAt      — 1 hour from now
// tokens.providerData   — see below
```

### `tokens.providerData` reference

| Key | Type | Description |
|-----|------|-------------|
| `email` | `string` | Google account email of the authorizing user |

### Step 3 — Refresh tokens

Google refresh tokens are long-lived (they only expire if revoked or unused for 6 months).

```typescript
const refreshed = await adapter.oauth.refreshToken(tokens.refreshToken!);
// refreshed.accessToken — new 1-hour token
// refreshed.refreshToken — same refresh token (non-rotating)
// refreshed.expiresAt   — 1 hour from now
```

## Real-time notifications (Pub/Sub)

Google Play delivers developer notifications via **Cloud Pub/Sub push subscriptions**. These are base64-encoded JSON messages, not standard HTTP webhook payloads.

### Setting up Pub/Sub notifications

1. In Google Play Console: **Monetize → Monetization Setup → Real-time developer notifications**
2. Enter your **Pub/Sub topic** (e.g. `projects/my-project/topics/play-notifications`)
3. In Cloud Console: Create a **push subscription** on the topic, targeting `https://yourapp.com/webhooks/google-play`

```typescript title="app/webhooks/google-play/route.ts" lineNumbers
import { NextRequest } from 'next/server';

export async function POST(req: NextRequest) {
  const rawBody = Buffer.from(await req.arrayBuffer());
  const headers = Object.fromEntries(req.headers.entries());

  const result = await hub.webhooks['google-play']({ rawBody, headers });
  return Response.json({ ok: true }, { status: result.statusCode ?? 200 });
}
```

### Notification event types

| Pub/Sub `notificationType` | `subscriptionNotification?` | Normalized `eventType` |
|---|---|---|
| SUBSCRIPTION_PURCHASED | `1` | `subscription.purchased` |
| SUBSCRIPTION_RENEWED | `2` | `subscription.renewed` |
| SUBSCRIPTION_CANCELLED | `3` | `subscription.cancelled` |
| SUBSCRIPTION_PURCHASED_FROM_FREE_TRIAL | `12` | `subscription.trial_converted` |
| SUBSCRIPTION_EXPIRED | `13` | `subscription.expired` |
| ONE_TIME_PRODUCT_PURCHASED | `-` | `purchase.completed` |
| ONE_TIME_PRODUCT_CANCELED | `-` | `purchase.cancelled` |

## App Store capability

<Callout type="info">
The Google Play adapter uses the **Edits API** pattern. All write operations (updating tracks, releases) open a draft edit, modify it, then commit. The adapter handles this automatically via the `withEdit()` helper.
</Callout>

```typescript
const store = hub.getAdapter('google-play').appStore!;
```

### App management

#### `getApp(tokens, packageName)`

Fetch app information. Use the **package name** (e.g. `com.example.myapp`) as the app ID.

```typescript
const app = await store.getApp(tokens, 'com.example.myapp');
// { id: 'com.example.myapp', name: 'My App', platform: 'android', ... }
```

<Callout type="warn">
`listApps` is **not supported** by the Google Play Developer API. Google Play API does not provide an endpoint to list all apps. You must store package names in your own database and pass them directly to capability methods.
</Callout>

### Build management

#### `listBuilds(tokens, packageName, opts?)`

List all App Bundles (AAB) and APKs uploaded to the app.

```typescript
const { items } = await store.listBuilds(tokens, 'com.example.myapp');
// items[0]:
// {
//   id: '42',              // versionCode as string
//   appId: 'com.example.myapp',
//   version: '2.4.1',      // versionName
//   buildNumber: '42',     // versionCode
//   type: 'bundle',        // 'bundle' | 'apk'
//   status: 'uploaded',
//   uploadedAt: Date
// }
```

#### `listBuildArtifacts(tokens, packageName, buildId)`

List AAB/APK artifacts for a specific versionCode.

```typescript
const artifacts = await store.listBuildArtifacts(tokens, 'com.example.myapp', '42');
// [{ id, buildId, type: 'aab' | 'apk', name, ... }]
```

### Release management

#### `listReleases(tokens, packageName, opts?)`

List releases across all tracks.

```typescript
const { items } = await store.listReleases(tokens, 'com.example.myapp', {
  track: 'production',               // Optional: 'production' | 'beta' | 'alpha' | 'internal'
  status: 'inProgress',             // Filter by status
});

// items[0]:
// {
//   id: 'production:42',
//   appId: 'com.example.myapp',
//   version: '2.4.1',
//   versionCodes: ['42'],
//   status: 'inProgress',
//   rolloutPercentage: 20,          // Rollout fraction if inProgress
//   createdAt: Date
// }
```

**Release status values:**

| Status | Description |
|--------|-------------|
| `draft` | Created but not yet published |
| `inProgress` | Being rolled out (partial percentage) |
| `halted` | Rollout paused |
| `completed` | Fully rolled out |

#### `getRelease(tokens, packageName, releaseId)`

Release IDs are formatted as `{track}:{versionCode}` (e.g. `production:42`).

```typescript
const release = await store.getRelease(tokens, 'com.example.myapp', 'production:42');
```

### Track management

Google Play has four standard release tracks: `production`, `beta`, `alpha`, and `internal`.

#### `listTracks(tokens, packageName)`

```typescript
const tracks = await store.listTracks(tokens, 'com.example.myapp');
// [
//   { id: 'com.example.myapp:production', name: 'production', type: 'production', releases: [...] },
//   { id: 'com.example.myapp:beta', name: 'beta', type: 'beta', releases: [...] },
//   ...
// ]
```

#### `getTrack(tokens, packageName, trackId)`

`trackId` can be the full `packageName:trackName` or just the track name.

```typescript
const track = await store.getTrack(tokens, 'com.example.myapp', 'production');
```

#### `updateTrack(tokens, packageName, trackId, update)`

Modify rollout state for releases on a track. **This commits the edit atomically.**

```typescript
// Increase rollout from 20% to 50%
await store.updateTrack(tokens, 'com.example.myapp', 'production', {
  rolloutPercentage: 50,
});

// Halt a rollout
await store.updateTrack(tokens, 'com.example.myapp', 'production', {
  halt: true,
});

// Resume a halted rollout
await store.updateTrack(tokens, 'com.example.myapp', 'production', {
  resume: true,
});

// Complete rollout (100%)
await store.updateTrack(tokens, 'com.example.myapp', 'production', {
  completeRollout: true,
});
```

### Publishing

#### `submitForReview(tokens, packageName, request)`

Submit the app for review (promote draft release to review state).

```typescript
const result = await store.submitForReview(tokens, 'com.example.myapp', {
  buildId: '42',     // versionCode
  track: 'production',
});
// { success: true, status: 'completed', message: '...', releaseId: '...' }
```

#### `releaseToTrack(tokens, packageName, request)`

Move a build to a specific track at a given rollout percentage.

```typescript
const result = await store.releaseToTrack(tokens, 'com.example.myapp', {
  buildId: '42',           // versionCode
  track: 'beta',
  rolloutPercentage: 100,
});
```

#### `haltRelease(tokens, packageName, releaseId, reason?)`

Emergency halt — sets all `inProgress` releases on the track to `halted`.

```typescript
await store.haltRelease(tokens, 'com.example.myapp', 'production:42', 'Critical ANR found');
```

### Android Vitals

#### `getVitalsSummary(tokens, packageName, opts?)`

Fetch performance metrics from the Play Developer Reporting API.

```typescript
const vitals = await store.getVitalsSummary(tokens, 'com.example.myapp', {
  metric: 'crash_rate',
  days: 28,
});
```

**Available `VitalMetricType` values:**

| Metric | Description |
|--------|-------------|
| `crash_rate` | Percentage of sessions with a crash |
| `anr_rate` | Application Not Responding (ANR) rate |
| `launch_time` | App startup time (P95 slow start rate) |
| `excessive_wakeups` | Excessive wakeup rate |
| `stuck_background_worker` | Stuck background wakelocks |

#### `listCrashClusters(tokens, packageName, opts?)`

```typescript
const { items } = await store.listCrashClusters(tokens, 'com.example.myapp', {
  limit: 10,
});
// items[0]:
// {
//   id, signature, count, impactedDevices,
//   exceptionType, stackTrace, latestOsVersion
// }
```

| Field | Value | Notes |
|-------|-------|-------|
| `expiresInSeconds` | `3600` | Access tokens expire after 1 hour |

## Error handling

| Scenario | Error class | Retryable |
|----------|-------------|-----------|
| Expired access token | `TokenRefreshError` | Yes |
| Package not found / access denied | `CapabilityError` | No |
| Rate limited (429) | `CapabilityError` | Yes |
| Edit conflict / stale edit | `CapabilityError` | Yes — retry |
| `listApps` called | `CapabilityError` | No — not supported |

## Common gotchas

**Edits API commits:**  Write operations (`updateTrack`, `releaseToTrack`, `submitForReview`) use `withEdit(..., { commit: true })` — they open an edit, apply changes, and commit in one atomic operation. Read operations open an edit then immediately delete it (no commit). If a write operation fails mid-flight, the adapter cleans up the draft edit automatically.

**`listApps` not supported:** Google Play API does not expose an apps list endpoint. Store package names in your own database and use them directly (e.g. `com.example.myapp`). The `getApp` method opens an edit to confirm the package is accessible.

**Pub/Sub message format:** Google sends base64-encoded JSON wrapped in a Pub/Sub envelope. The adapter unwraps the envelope and parses the inner `developerNotification` payload automatically.

**Track promotion:** There is no dedicated "promote from beta to production" endpoint. Create a new production release with the same versionCode(s) via `releaseToTrack` with `track: 'production'`.

**Review replies:** You can only reply to reviews that have `replyAllowed: true` (most do). New reviews take a short time to become replyable after submission.
