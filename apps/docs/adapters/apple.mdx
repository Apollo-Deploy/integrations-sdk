## Overview

The Apple App Store Connect adapter provides comprehensive access to the **App Store Connect API** for iOS, macOS, tvOS, and watchOS apps. It authenticates using a **short-lived JWT** signed with your private key (ES256) — no OAuth flow, no user authorization. Access is granted via API keys created in App Store Connect.

**Capability:** [`app-store`](/docs/capabilities/app-store)

**Auth model:** JWT (API key-based) — no user OAuth

**Token lifetime:** 20 minutes maximum per JWT (fresh token generated per request)

---

## Prerequisites

1. Go to [appstoreconnect.apple.com](https://appstoreconnect.apple.com) → **Users and Access → Keys**
2. Click **"+"** to generate a new API key
3. Choose a **Role** (Admin for full access, or a more restricted role)
4. Download the **`.p8` private key** file — you can only download it **once**
5. Note the **Key ID** and **Issuer ID** from the keys page
6. For the vendor number (financial reports): go to **Payments and Financial Reports → your vendor number**
7. For App Store Server Notifications: set your URL in your app's **App Store → App Information → App Store Server Notifications**

## Configuration

```typescript title="lib/integrations.ts" lineNumbers
import { createAppleAdapter } from '@apollo-deploy/adapter-apple';

const apple = createAppleAdapter({
  issuerId: process.env.APPLE_ISSUER_ID!,
  keyId: process.env.APPLE_KEY_ID!,
  privateKey: process.env.APPLE_PRIVATE_KEY!,
  vendorNumber: process.env.APPLE_VENDOR_NUMBER!,
  webhookSecret: process.env.APPLE_WEBHOOK_SECRET,
});
```

### Configuration options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `issuerId` | `string` | ✅ | Issuer ID — UUID from App Store Connect → Users and Access → Keys |
| `keyId` | `string` | ✅ | Key ID — 10-character string shown next to your API key |
| `privateKey` | `string` | ✅ | Full contents of the `.p8` file including PEM headers |
| `vendorNumber` | `string` | ✅ | Vendor number for financial/sales reports |
| `webhookSecret` | `string?` | — | Shared secret (password) configured in App Store Server Notifications |

```bash title=".env"
APPLE_ISSUER_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
APPLE_KEY_ID=ABCDE12345

# Full .p8 content with literal \n newlines

# Optional: for App Store Server Notifications

```

<Callout type="warn">
The `.p8` private key can only be downloaded once from App Store Connect. If lost, you must revoke the key and create a new one (invalidating any existing API connections).
</Callout>

**Loading the key from a file:**

```typescript
import { readFileSync } from 'node:fs';

const apple = createAppleAdapter({
  issuerId: process.env.APPLE_ISSUER_ID!,
  keyId: process.env.APPLE_KEY_ID!,
  privateKey: readFileSync('./keys/AuthKey_ABCDE12345.p8', 'utf8'),
  vendorNumber: process.env.APPLE_VENDOR_NUMBER!,
});
```

## Authentication

App Store Connect uses **JSON Web Tokens (JWT)** signed with ES256 (ECDSA + SHA-256). A new JWT is generated automatically when an existing token is within 60 seconds of expiry.

**JWT claims:**

| Claim | Value |
|-------|-------|
| `iss` | `issuerId` |
| `iat` | Current Unix timestamp |
| `exp` | `iat + 1200` (20 minutes max) |
| `aud` | `appstoreconnect-v1` |

The JWT is sent as `Authorization: Bearer <jwt>` on every request.

<Callout type="info">
There is no OAuth flow for Apple App Store Connect. The `oauth` handler exists for interface compatibility but generates a JWT from your private key rather than redirecting a user. You call `adapter.oauth.exchangeCode({ code: '' })` to get an initial `TokenSet` — store this and use it for all API calls.
</Callout>

## App Store Server Notifications

Apple sends server-to-server notifications for in-app purchase events. These are payload-level `signedPayload` JWTs verified against Apple's root certificates.

```typescript title="app/webhooks/apple/route.ts" lineNumbers
import { NextRequest } from 'next/server';

export async function POST(req: NextRequest) {
  const rawBody = Buffer.from(await req.arrayBuffer());
  const headers = Object.fromEntries(req.headers.entries());

  const result = await hub.webhooks.apple({ rawBody, headers });
  return Response.json({ ok: true }, { status: result.statusCode ?? 200 });
}
```

### Supported notification types

| Notification `notificationType` | Normalized `eventType` | Description |
|---|---|---|
| `DID_RENEW` | `subscription.renewed` | Subscription successfully renewed |
| `SUBSCRIBED` | `subscription.started` | New subscription |
| `DID_CHANGE_RENEWAL_STATUS` | `subscription.renewal_status_changed` | Auto-renewal toggled |
| `DID_FAIL_TO_RENEW` | `subscription.renewal_failed` | Billing failure |
| `EXPIRED` | `subscription.expired` | Subscription fully expired |
| `REFUND` | `subscription.refunded` | Purchase refunded |
| `CONSUMPTION_REQUEST` | `subscription.consumption_request` | Consumable refund prompt |
| `REVOKE` | `entitlement.revoked` | Family Sharing revocation |

### App management

#### `listApps(tokens)`

List all apps in your App Store Connect account.

```typescript

// items[0]:
// { id, name, bundleId, platform, primaryLocale, createdAt }
```

```typescript

```

|--------|-------------|

### Track and version management

#### `listTracks(tokens, appId)`

Returns the two App Store tracks: `app_store` (production) and `testflight` (beta).

```typescript
const tracks = await store.listTracks(tokens, '1234567890');
// [
//   { id: 'app_store', name: 'App Store', type: 'production', releases: [...] },
//   { id: 'testflight', name: 'TestFlight', type: 'beta', releases: [...] }
// ]
```

#### `updateTrack(tokens, appId, trackId, update)`

Control phased releases on the App Store track.

```typescript
// Complete a phased rollout (release to 100%)
await store.updateTrack(tokens, '1234567890', 'app_store', {
  completeRollout: true
});

// Pause a phased rollout
await store.updateTrack(tokens, '1234567890', 'app_store', {
  halt: true
});

// Resume a paused rollout
await store.updateTrack(tokens, '1234567890', 'app_store', {
  resume: true
});
```

| Rate limited (429) | `CapabilityError` | Yes |

| Phased release not found | `CapabilityError` | No |

## Common gotchas

**App Store Connect API keys have roles:** Choose the appropriate role. `Customer Support` can respond to reviews. `Developer` can manage builds and testing. `Admin` has full access.

**Phased release requires a prior setup:** You can only control a phased release if `appStoreVersionPhasedRelease` was set up during the initial version creation. If it doesn't exist, `updateRollout` / `updateTrack` will throw a `CapabilityError`.

**JWT expiry limit:** Apple enforces a maximum JWT lifetime of **20 minutes** (`exp - iat ≤ 1200`). The adapter generates fresh JWTs with this maximum.

**`listApps` limitation:** The Apps endpoint returns all apps associated with your API key's team. If your key is scoped to a specific app, `listApps` may return only that app.

**Reviews are per-territory:** `listReviews` returns reviews from all territories by default. Use the `territory` parameter to filter to a specific App Store storefront (e.g. `'GBR'`, `'USA'`, `'JPN'`).
