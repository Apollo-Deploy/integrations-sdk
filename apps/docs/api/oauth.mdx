## `OAuthHandler`

The contract every adapter's OAuth implementation must satisfy. Access via `adapter.oauth`.

### `getAuthorizationUrl(params)`

```typescript
getAuthorizationUrl(params: AuthorizationParams): string
```

Builds the provider authorization URL. The hub manages state signing/validation — the adapter only constructs the URL.

**`AuthorizationParams`**

### `exchangeCode(params)`

```typescript
exchangeCode(params: CodeExchangeParams): Promise<TokenSet>
```

Exchanges the authorization code for tokens.

**`CodeExchangeParams`**

<TypeTable
  type={{
    code: { description: 'Authorization code from the provider callback.', type: 'string' },
    redirectUri: { description: 'Must exactly match the redirectUri used in getAuthorizationUrl.', type: 'string' },
    codeVerifier: { description: 'PKCE code verifier, if used.', type: 'string | undefined' },
  }}
/>

### `refreshToken(refreshToken)`

```typescript
refreshToken(refreshToken: string): Promise<TokenSet>
```

Refreshes an expired access token. Throws `OAuthError` if the provider does not support refresh.

### `getIdentity(accessToken)`

```typescript
getIdentity(accessToken: string): Promise<ProviderIdentity>
```

Fetches the authenticated user's identity from the provider. Used to populate connection metadata. Throws `OAuthError` with code `OAUTH_IDENTITY_FAILED` if the provider returns an HTTP error (e.g. invalid or expired token).

### `afterAuthorize(tokens)` (optional)

```typescript
afterAuthorize?(tokens: TokenSet): Promise<PostAuthResult>
```

Optional post-authorization hook. Used to discover provider-specific resources after the OAuth flow:
- **GitHub**: discover installations and `installationId`
- **Jira**: discover accessible cloud sites and `cloudId`

Returns `null` if not applicable.
