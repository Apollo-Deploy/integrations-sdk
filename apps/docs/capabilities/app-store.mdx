The `app-store` capability provides a unified interface for mobile app distribution platforms. Adapters that declare it expose an `appStore` property implementing the full `AppStoreCapability` interface.

**Adapters:** [Apple App Store Connect](/docs/adapters/apple), [Google Play Console](/docs/adapters/google-play)

## Accessing the capability

```typescript title="lib/app-store.ts" lineNumbers
import { hub } from '@/lib/integrations';

const apple = hub.getAdapter('apple');
const store = apple.appStore!;
```

## 1. App management

```typescript lineNumbers
const apps = await store.listApps(tokens);
const app = await store.getApp(tokens, 'com.example.myapp');
// app.id, app.bundleId, app.name, app.platform
```

<Callout type="warn">
Google Play does not support `listApps` — it throws `CapabilityError`. Store package names in your database.
</Callout>

## 2. Builds & artifacts

```typescript lineNumbers
const builds = await store.listBuilds(tokens, appId, {
  version: '2.1.0',
  status: 'valid',  // 'processing' | 'valid' | 'invalid' | 'expired'
  limit: 20,
});
const build = await store.getBuild(tokens, appId, buildId);
// build.id, build.version, build.buildNumber, build.status, build.uploadedAt
```

### Artifacts

Download dSYMs (Apple) or ProGuard mapping files (Google):

```typescript lineNumbers
const artifacts = await store.listBuildArtifacts(tokens, appId, buildId);
// artifacts: StoreArtifact[] — id, type, fileName, size, downloadable

const { url, expiresAt } = await store.getArtifactDownloadUrl(tokens, appId, buildId, artifactId);
```

### Binary uploads

Upload AAB, APK, or IPA files to the store:

```typescript lineNumbers
// Google Play — store channel (default)
const result = await store.uploadBinary(tokens, appId, aabFile, {
  fileType: 'aab',
  channel: 'store',
  commitEdit: true,
});
// { channel: 'store', fileType: 'aab', build: StoreBuild }

// Google Play — internal sharing
const result = await store.uploadBinary(tokens, appId, apkFile, {
  channel: 'internal-sharing',
});
// { channel: 'internal-sharing', fileType: 'apk', artifact: InternalSharingArtifact }
```

**Provider support matrix:**

| fileType | channel | Google Play | Apple |
|----------|---------|-------------|-------|
| `aab` | `store` | ✓ Edits API | ✗ |
| `apk` | `store` | ✓ Edits API | ✗ |
| `ipa` | `store` | ✗ | ✗ (use `xcrun altool`) |
| `aab`/`apk` | `internal-sharing` | ✓ IAS API | ✗ |

## 3. Releases

```typescript lineNumbers
const releases = await store.listReleases(tokens, appId, { status: 'rolling_out' });
```

## 4. Tracks & versions

```typescript lineNumbers
const tracks = await store.listTracks(tokens, appId);
// Apple: ['app_store', 'testflight']
// Google: ['production', 'beta', 'alpha', 'internal']

const track = await store.getTrack(tokens, appId, 'production');

await store.updateTrack(tokens, appId, 'production', {
  rolloutPercentage: 50,        // Set rollout %
  halt: true,                   // Halt rollout
  resume: true,                 // Resume halted rollout
  completeRollout: true,        // Release to 100%
  releases: [{ versionCodes: ['42'], status: 'completed' }],
});
```

### Versions

```typescript lineNumbers
const versions = await store.listVersions(tokens, appId);
const version = await store.getVersion(tokens, appId, versionId);
```

## 5. Publishing

### Submit for review

```typescript lineNumbers
const result = await store.submitForReview(tokens, appId, {
  versionId: '1.2.3',
  releaseNotes: [{ language: 'en-US', text: 'Bug fixes and improvements.' }],
  phasedRelease: true,     // Apple: enable phased release
  autoRelease: false,      // Apple: manual release after approval
});
// result: { success, status, message, releaseId }
```

### Release to track

Release an approved version to users (supports staged rollout):

```typescript lineNumbers
await store.releaseToTrack(tokens, appId, {
  buildId: '42',
  track: 'production',
  rolloutPercentage: 10,
  releaseNotes: [{ language: 'en-US', text: 'New features!' }],
});
```

### Update rollout / halt

```typescript lineNumbers
await store.updateRollout(tokens, appId, releaseId, { rolloutPercentage: 50 });
await store.updateRollout(tokens, appId, releaseId, { completeRollout: true });

await store.haltRelease(tokens, appId, releaseId, 'Critical issue found');
```

## 6. Reviews & ratings

```typescript lineNumbers
  territory: 'US',
  minRating: 1,
  maxRating: 2,

// Note: Google Play throws CapabilityError for deleteReviewReply

const summary = await store.getRatingSummary(tokens, appId, { territory: 'US' });
// summary.averageRating, summary.totalRatings, summary.histogram

const ratings = await store.listRatings(tokens, appId, { territory: 'US', limit: 100 });
```

## 8. Vitals & crash reporting

```typescript lineNumbers
// Summary across all metrics
const vitals = await store.getVitalsSummary(tokens, appId, {
  version: '2.1.0',
  startDate: new Date('2025-01-01'),
  endDate: new Date('2025-01-31'),
});
// vitals.metrics.crashRate, vitals.metrics.anrRate, vitals.metrics.launchTime

// Detailed metric with data points
const metric = await store.getVitalMetric(tokens, appId, 'crash_rate', {
  granularity: 'daily',
});
// metric.dataPoints: [{ timestamp, value }]

// Crash clusters
const crashes = await store.listCrashClusters(tokens, appId, { version: '2.1.0', limit: 10 });
const cluster = await store.getCrashCluster(tokens, appId, clusterId);

// ANR clusters (Android only — Apple returns empty array)
const anrs = await store.listAnrClusters(tokens, appId, { limit: 10 });
```

**Available `VitalMetricType` values:** `crash_rate`, `anr_rate`, `excessive_wakeups`, `stuck_background_worker`, `launch_time`, `render_time`, `permission_denial_rate`, `battery_drain`, `memory_usage`, `disk_usage`, `hang_rate`, `scroll_hitch_rate`, `disk_writes`

## 10. App Recovery (Google Play only)

Remotely remediate issues for users on affected app versions. Apple throws `CapabilityError` for all recovery methods.

```typescript lineNumbers
// List existing recovery actions
const actions = await store.listAppRecoveryActions(tokens, appId, { versionCode: '42' });

// Create a new recovery action (DRAFT status)
const action = await store.createAppRecoveryAction(tokens, appId, {
  versionCodes: ['40', '41'],
  remediationType: 'REMOTE_IN_APP_UPDATE', // or 'FORCE_CRASH'
});

// Deploy a draft recovery
await store.deployAppRecoveryAction(tokens, appId, action.appRecoveryId);

// Cancel a recovery
await store.cancelAppRecoveryAction(tokens, appId, action.appRecoveryId);

// Expand targeting
await store.addAppRecoveryTargeting(tokens, appId, action.appRecoveryId, {
  versionCodes: ['42'],
  regions: { includeList: ['US', 'CA'] },
});
```

## 11. Phased releases

Control gradual rollout of app versions.

```typescript lineNumbers
// Enable phased release for a version
const phased = await store.createPhasedRelease(tokens, appId, {
  versionId: 'version-uuid',     // Apple: version UUID, Google: 'track:versionCode'
  rolloutPercentage: 1,          // Google only — Apple uses fixed 7-day schedule
});

// Check current state
const current = await store.getPhasedRelease(tokens, appId, versionId);

// Pause / resume / complete
await store.updatePhasedRelease(tokens, appId, phasedId, { state: 'paused' });
await store.updatePhasedRelease(tokens, appId, phasedId, { state: 'active' });
await store.updatePhasedRelease(tokens, appId, phasedId, { state: 'complete' });

// Google only: set exact percentage
await store.updatePhasedRelease(tokens, appId, phasedId, { rolloutPercentage: 50 });

// Cancel a planned phased release
await store.deletePhasedRelease(tokens, appId, phasedId);
```

**Platform differences:**

| Behavior | Apple | Google Play |
|----------|-------|-------------|
| Rollout schedule | Fixed 7 days (1%→2%→5%→10%→20%→50%→100%) | Manual — set any percentage |
| Custom percentage | ✗ Ignored | ✓ `rolloutPercentage` |
| `currentDayNumber` | ✓ (0–7) | ✗ `undefined` |

## 12. Manual release requests

Trigger an immediate release for a version pending developer approval.

```typescript lineNumbers
const request = await store.createReleaseRequest(tokens, appId, versionId);
// { id, versionId, requestedAt: Date }
```

Apple: Releases a "Pending Developer Release" version — **irreversible**.
Google Play: Commits the production track with status `completed`.
