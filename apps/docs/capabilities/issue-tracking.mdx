title: Issue Tracking
description: List, create, update, and comment on issues across Jira, GitLab, and Linear.
---

The `issue-tracking` capability provides a unified interface for project management tools. Adapters that declare it expose an `issueTracking` property.

**Adapters:** [Jira](/docs/adapters/jira), [GitLab](/docs/adapters/gitlab), [Linear](/docs/adapters/linear)

## Accessing the capability

```typescript title="lib/issues.ts" lineNumbers
import { hub } from '@/lib/integrations';

const jira = hub.getAdapter('jira');
const it = jira.issueTracking!;
```

## Methods

### `listProjects`

List the projects/boards/teams the user has access to.

```typescript lineNumbers
const projects = await it.listProjects(tokens);
// projects: ProjectRef[]  — project.id, project.key, project.name
```

### `listIssues`

Fetch a filtered list of issues.

```typescript lineNumbers
const issues = await it.listIssues(tokens, {
  projectId: 'PROJ-001',
  status: 'open',
  assigneeId: 'user-123',
  labels: ['bug', 'deploy-blocker'],
  page: 1,
  perPage: 50,
});
```

### `getIssue`

Fetch a single issue by its provider-specific ID.

```typescript lineNumbers
const issue = await it.getIssue(tokens, 'PROJ-42');
// issue.id, issue.title, issue.status, issue.assignee, issue.labels
```

### `createIssue`

Create a new issue.

```typescript lineNumbers
const issue = await it.createIssue(tokens, {
  projectId: 'PROJ',
  title: 'Deploy #1234 reverted — investigation needed',
  description: 'Deployment was reverted due to increased error rate.\n\nDeploy URL: ...',
  labels: ['incident', 'deploy'],
  priority: 'high',
  assigneeId: 'user-abc',
});
```

### `updateIssue`

Update an existing issue's fields or status.

```typescript lineNumbers
const updated = await it.updateIssue(tokens, 'PROJ-42', {
  status: 'done',
  title: 'Deploy #1234 reverted — resolved',
});
```

### `addComment`

Post a comment on an issue.

```typescript lineNumbers
const comment = await it.addComment(
  tokens,
  'PROJ-42',
  'Deploy has been re-queued. Will retry in 10 minutes.',
);
// comment.id, comment.body, comment.author, comment.createdAt
```

## Error handling

All issue tracking methods throw `CapabilityError` on failure. The `retryable` flag is `true` for HTTP `429` (rate limit) and `5xx` (server error) responses. See the [Jira](/docs/adapters/jira#error-handling), [GitLab](/docs/adapters/gitlab#error-handling), and [Linear](/docs/adapters/linear#error-handling) adapter docs for full error tables.

## Types

  id: string;

  description?: string;
  status: string;
  priority?: string;
  labels: string[];

  projectId: string;
  createdAt: Date;
  updatedAt: Date;

}

interface IssueComment {
  id: string;
  body: string;
  author: ActorInfo;
  createdAt: Date;
}

interface ProjectRef {
  id: string;
  key: string;
  name: string;
}
