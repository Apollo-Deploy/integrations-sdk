## Installation

```bash title="Terminal"
bun add @apollo-deploy/adapter-discord
```

## Configuration

```typescript title="lib/integrations.ts" lineNumbers
import { createDiscordAdapter } from '@apollo-deploy/adapter-discord';

const discord = createDiscordAdapter({
  clientId: process.env.DISCORD_CLIENT_ID!,
  clientSecret: process.env.DISCORD_CLIENT_SECRET!,
  botToken: process.env.DISCORD_BOT_TOKEN!,
  publicKey: process.env.DISCORD_PUBLIC_KEY!,
});
```

| Option | Type | Description |
|--------|------|-------------|
| `clientId` | `string` | Discord application Client ID |
| `clientSecret` | `string` | Discord application Client Secret |
| `botToken` | `string` | Bot token (`Bot xxxxxxxxxxx`) |
| `publicKey` | `string` | Ed25519 public key for interaction signature verification |

## Environment variables

## OAuth

```typescript title="lib/discord-oauth.ts" lineNumbers
const url = adapter.oauth.getAuthorizationUrl({
  state,
  scopes: ['bot', 'applications.commands'],
  redirectUri,
  extras: { permissions: '2048' }, // SEND_MESSAGES permission bit
});

const tokens = await adapter.oauth.exchangeCode({ code, redirectUri });
// tokens.providerData.guildId
// tokens.providerData.guildName
```

<Callout type="info">
Discord bot tokens do not expire. `expiresInSeconds` is `null`.
</Callout>

## Webhook events

Discord uses Ed25519 signature verification. The adapter verifies `X-Signature-Ed25519` and `X-Signature-Timestamp` headers.

Discord does not send a delivery ID — the adapter generates one from a body hash for idempotency.

| Discord event | Normalized `eventType` |
|---|---|
| `MESSAGE_CREATE` | `message.created` |
| `MESSAGE_UPDATE` | `message.updated` |
| `MESSAGE_DELETE` | `message.deleted` |
| `MESSAGE_REACTION_ADD` | `reaction.added` |
| `MESSAGE_REACTION_REMOVE` | `reaction.removed` |
| `GUILD_MEMBER_ADD` | `channel.member_joined` |

## Token metadata

| Field | Value |
|-------|-------|
| `expiresInSeconds` | `null` (never) |
| `refreshable` | `false` |
| `rotatesRefreshToken` | `false` |
| `requiresRefreshLock` | `false` |
