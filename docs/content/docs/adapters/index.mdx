## Token lifecycle

Each adapter declares a `tokenMetadata` object that drives the hub's refresh scheduler:

| Field | Type | Description |
|-------|------|-------------|
| `expiresInSeconds` | `number \| null` | `null` = token never expires |
| `refreshable` | `boolean` | Whether a refresh flow exists |
| `rotatesRefreshToken` | `boolean` | Refresh tokens rotate on each use (requires atomic swap) |
| `requiresRefreshLock` | `boolean` | Distributed lock must be held during refresh |

## Listing registered adapters

Call `hub.listAdapters()` to get a snapshot of every registered adapter, including its metadata:

```typescript title="lib/integrations.ts" lineNumbers
const adapters = hub.listAdapters();
// [
//   {
//     key: 'github',
//     id: 'github',
//     name: 'GitHub',
//     capabilities: ['source-control'],
//     metadata: {
//       description: 'Connect GitHub repositories...',
//       category: 'Source Control',
//       dateAdded: '2024-01-15',
//       websiteUrl: 'https://github.com',
//       docsUrl: 'https://docs.github.com/en/apps',
//     },
//   },
//   ...
// ]
```
