## `defineAdapter<TConfig>(definition)`

Creates a type-safe adapter factory function from a definition object.

```typescript
function defineAdapter<TConfig>(
  definition: AdapterDefinition<TConfig>
): (config: TConfig) => IntegrationAdapter<TConfig>
```

The returned function (`createXxxAdapter`) takes your provider-specific config object and produces a fully-constructed `IntegrationAdapter`. Capabilities are only wired if they are both declared in `capabilities` **and** have a corresponding factory function.

```typescript title="src/adapter.ts" lineNumbers
export const createMyAdapter = defineAdapter<MyConfig>({
  id: 'my-provider',
  name: 'My Provider',
  capabilities: ['messaging'] as const,
  tokenMetadata: { expiresInSeconds: 3600, refreshable: true, rotatesRefreshToken: false, requiresRefreshLock: false },
  createOAuthHandler: (config) => createMyOAuth(config),
  createWebhookHandler: (config) => createMyWebhook(config),
  createMessaging: (config) => createMyMessaging(config),
});

// Usage:
const adapter = createMyAdapter({ botToken: '...', signingSecret: '...' });
```

## Capability wiring

The factory automatically wires capabilities based on `capabilities` declarations:

```typescript
// Only wired if 'source-control' is in capabilities AND createSourceControl is provided
adapter.sourceControl = capabilities.includes('source-control')
  ? createSourceControl(config)
  : undefined;
```

This means `adapter.sourceControl` is `undefined` if you forgot to add `'source-control'` to `capabilities`, even if you provided `createSourceControl`. TypeScript will warn about any mismatch.
